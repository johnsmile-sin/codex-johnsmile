# stock-analysis-core

## 목적

단일 종목에 대해 **경량 종합 분석 리포트**를 생성합니다.  
`analysis/stock_report.py`의 `generate_report()` 함수와 동일한 7개 섹션 구조를 따릅니다.

> **선행 조건:** `stock-risk-guard` Skill이 반드시 함께 로드되어야 합니다.  
> 이 Skill 단독으로는 실행할 수 없습니다.

---

## 트리거 조건

다음 중 하나에 해당하면 이 Skill이 활성화됩니다.

| 트리거 | 예시 |
|--------|------|
| 종목명 + 분석 요청 | "삼성전자 분석해줘", "현대차 리포트" |
| 종목코드 + 분석 요청 | "005930 분석", "000660 리포트 뽑아줘" |
| CLI 명령 | `analyze_stock`, `today_candidates` |
| 비교 분석 | "삼성전자 vs SK하이닉스" → 각 종목 개별 리포트 2건 생성 |

## 입력 스펙

```
입력 유형: 자연어 또는 CLI 인수
필수:
  - stock_name  (종목명, 부분 일치 허용)  또는
  - stock_code  (6자리 종목코드)
선택:
  - analysis_date  (기준일, 미입력 시 오늘 날짜)
```

```bash

# 자연어

```

## 분석 파이프라인

```
입력 (종목명/코드)
      │
      ▼
[1] 종목 조회 (services/market_data.py)
      │  실패 → "조회 불가 종목" 안내 후 종료
      ▼
[2] 스캐너 점수 계산 (strategy/scanner.py)
      │  결과: score (0~100), decision (관심/관찰/보류/제외)
      ▼
[3] 뉴스 감성 수집 (services/news_data.py)
      │  결과: 긍정/중립/부정 건수, 최근 뉴스 3건
      ▼
[4] 리포트 생성 (analysis/stock_report.py → generate_report())
      │  결과: 7개 섹션 dict
      ▼
[5] stock-risk-guard 후처리
      │  금지 항목 제거, 면책 문구 삽입
      ▼
[6] 최종 출력 (format_report_text() 또는 Streamlit UI)
```

## 7개 섹션 출력 구조

### 섹션 1 — 기본 정보

| 항목 | 설명 |
|------|------|
| 종목코드 | 6자리 코드 |
| 종목명 | 한국어 공식 명칭 |
| 시장 | KOSPI / KOSDAQ |
| 섹터 | 반도체, 바이오, 금융 등 |
| 현재가 | 원화 (쉼표 포맷) |
| 전일 대비 | 등락률 (부호 포함) |
| 거래대금 | 억원 단위 |

### 섹션 2 — 기술적 점수

스캐너 10개 규칙 기반 (0~100점):

| 규칙 | 배점 |
|------|------|
| 거래대금 100억 이상 | +20 |
| 거래량 20일 평균 2배↑ | +20 |
| 종가 > MA5 | +15 |
| 종가 > MA20 | +15 |
| MA5 > MA20 (정배열) | +10 |
| 양봉 마감 | +10 |
| 뉴스 1건 이상 | +10 |
| 등락률 20% 이상 (급등 위험) | -15 |
| 부채비율 200% 이상 | -10 |
| PER 또는 ROE 음수 | -10 |

출력 항목: 스캐너 점수, 스캐너 판단, MA5/MA20/MA60, 정배열 여부, 거래량 비율

### 섹션 3 — 재무 요약

| 항목 | 판단 기준 |
|------|-----------|
| PER | 0 미만 → 적자 경고 / 50 초과 → 고평가 경고 |
| PBR | 1.0 미만 → 저평가 가능성 |
| ROE | 15% 이상 → 우량, 0 미만 → 적자 경고 |
| 부채비율 | 200% 이상 → 재무 리스크 |
| 재무 리스크 여부 | 위 4개 조건 중 하나라도 해당 시 `true` |

### 섹션 4 — 뉴스 감성 요약

- 긍정 / 중립 / 부정 건수 및 비율
- 최근 뉴스 3건 (날짜, 제목, 감성, 영향도)
- 감성 우위 판단 기준:
  - 긍정 50% 이상 → 긍정 우위
  - 부정 40% 이상 → 부정 우위
  - 그 외 → 중립

### 섹션 5 — 최종 투자 판단

**판정 5단계:**

| 판정 | 색상 코드 | 발동 조건 |
|------|-----------|-----------|
| 적극 매수 | `#1A5276` (남색) | 점수 80↑ + 긍정 뉴스 + 재무 리스크 없음 |
| 분할 매수 | `#27AE60` (초록) | 점수 60↑ + 부정 뉴스 없음 |
| 관망 | `#F39C12` (주황) | 점수 40~79 또는 부정/재무 리스크 있음 |
| 비중 축소 | `#E67E22` (짙은 주황) | 점수 40~59 + 부정 뉴스 우위 |
| 매도 | `#E74C3C` (빨강) | 점수 40 미만 + 부정 뉴스 + 재무 리스크 |

**판정별 필수 포함 항목:**

```
목표 수익률  — 판정이 적극 매수/분할 매수일 때만 (예: "+12% 내외")
손절 라인   — 판정이 적극 매수/분할 매수일 때 (MA20 기준 또는 현재가 -7%)
진입 타이밍 — 모든 판정에서 한 문장으로 안내
핵심 근거   — 3가지 (스캐너 이유 → 재무 → 뉴스 순으로 보완)
리스크      — 3가지 (스캐너 리스크 → 재무 → 뉴스 → 공통 순으로 보완)
```

### 섹션 6 — 한 줄 결론

판정에 맞게 종목명·점수·감성을 조합한 요약 문장 1개.

```
예시:
  삼성전자은 기술적 흐름(65점)이 양호하여 리스크 분산을 위한 분할 매수 접근이 적합합니다.
  에코프로는 점수(32점) 및 부정 뉴스 신호로 보유 시 분할 매도가 적합합니다.
```

### 섹션 7 — 메타

```
생성일      : YYYY-MM-DD
데이터 신뢰도: 낮음 (Mock) / 보통 (지연 15분) / 높음 (실시간)
주의사항    : "본 리포트는 투자 참고용 자료입니다. ..."
```

## 비교 분석 출력 규칙

두 종목 비교 요청 시 (`A vs B` 형태) 아래 순서로 출력합니다.

```
1. 종목 A 개별 리포트 (전체 템플릿)
2. 종목 B 개별 리포트 (전체 템플릿)
3. 비교 요약 테이블

| 항목         | 종목 A  | 종목 B  | 우위   |
|-------------|---------|---------|--------|
| 스캐너 점수  | XX점    | XX점    | A/B    |
| PER         | XX.X배  | XX.X배  | A/B    |
| ROE         | XX.X%   | XX.X%   | A/B    |
| 부채비율     | XXX%    | XXX%    | A/B    |
| 뉴스 긍정 비율| XX%   | XX%     | A/B    |
| 최종 판정    | [판정]  | [판정]  | —      |
```

## 오류 처리

| 상황 | 응답 |
|------|------|
| 종목 조회 실패 | "❌ '[입력값]' 종목을 찾을 수 없습니다. 종목명 또는 6자리 코드를 확인하세요." |
| 데이터 부족 (H-01~H-04) | stock-risk-guard의 판단 보류 규칙 적용 |
| 입력값 없음 | "종목명 또는 종목코드를 입력해주세요. 예: 삼성전자, 005930" |
| 복수 종목 매칭 | 첫 번째 일치 항목 사용 후 "입력과 유사한 종목: [목록]" 안내 |

## 사용 제한

- 이 Skill은 **분석 참고용** 데이터만 제공합니다.
- 실거래 주문, 자동 매매, 포트폴리오 리밸런싱 기능은 포함하지 않습니다.
- `stock-risk-guard` 없이 단독 실행 시 경고 메시지를 출력하고 분석을 중단합니다.

```
⚠️ stock-risk-guard Skill이 로드되지 않았습니다.
   안전 가드레일 없이는 분석을 실행하지 않습니다.
   stock-risk-guard를 먼저 로드하세요.
```
